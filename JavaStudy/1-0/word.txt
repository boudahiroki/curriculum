ITについて
IT
→Information Technologyの略称。情報技術。コンピュータやネットワークといった情報勝利に関する技術の総称

ICT
→Information and Communication Technologyの略称。情報伝達技術や情報通信技術。「人と人」「人ともの」それぞれの間で情報や知識を共有することでコミュニケーションがより強調されている。

IoT
→Internet of Thingsの略称。ものとインターネット。PCやスマホなどの従来機器を除いたあらゆるものがインターネットとつながる仕組みや技術のこと


OSについて
OS
→オペレーティングシステムの略称。コンピュータ全体の管理、制御をし人が使えるようにする

Windows
→マイクロソフト社が開発したOS。最新はWiodows10。ウィンドウを開いて操作することが語源となっている。

MacOS
→apple社が開発したOS。MacはMacintoshの略称。

UNIX
→AT＆T社が開発したOS。現存するOSの中で最も古い。コマンド画面での操作が基本となる。知的財産権が企業に属しているため使用には費用がかかる。

Linux
→フィンランドの大学生がUNIXを参考にして独自開発したOS。無料で使える。

言語・環境
HTML
→ハイパーテキスト・マークアップ・ランゲージの略称。Webページを作る最も基本的な言語。ほとんどのWebページがこの言語で作られている

CSS
→カスケーティング・スタイル・シートの略称。Webページを装飾する。HTMLと組み合わせて使う言語。

JavaScript
→Webページに動きをつける。

.NET(C#,VB)
→microsoftが開発した言語。windows7以降では標準でインストールされている
VB（VisualBasic）とC#は異なる言語だが.NET Framework環境で使用可能

Java
→世界で最も使われている言語の1つ。汎用性が高くMacOSやWindowsといったOSに依存せず動作するため人気が高い。

Java SE
→Java platform Standard Editionの略称。Javaの基本的なAPIをまとめたもの。

Java EE
→Java Platform Enterprise Editionの略称。大規模なシステム開発する場合にJava SEと合わせて使われる。

Perl
→歴史の長い言語。テキストの処理が得意。利用者が多く参考にできるプログラムが多いためプログラムが作りやすい。

PHP
→動的なWebページを作成できるサーバーサイド言語。比較的文法が簡単で習得しやすい。

Ruby
→動的なWebページを作成できるサーバーサイド言語。日本人によって開発された。Ruby on Railsというフレームワークを使うことで効率よくシステムを作ることができる。

Apache
→世界で最も使用されているサーバーソフトウェア。レンタルサーバーに最初から組み込まれている場合が多い。情報も多いのでトラブルの解決がしやすい

Nginx
→apacheに次いでシェアが多い。高速で大量の処理が得意。動的処理は苦手なため動画を中心としたサイトの場合apacheの方が良い。

IIS
→windowsで動作するため使いやすい。マウスでの操作ができるため視覚的に操作ができる。

SQL
→シーケルと呼ぶ。データベース言語で最も普及している。データベースを操作できるようにしたものがデータベース言語。
オブジェクトを定義するデータ定義言語、データを操作するデータ操作言語、データへのアクセスを制御するデータ制御言語の3つの言語から構成されている。

DB2
→リレーショナルデータベース管理システム。言語はSQL。管理ツールや開発ツールの製品群。

Mysql
→リレーショナルデータベース。言語はSQL。無料で使用できる。多くのOSに対応している。操作方法がシンプル

MariaDB
→Mysqlから派生したリレーショナルデータベース。無料で使用できる。開発者はMysqlと同じだが完全なオープンソースなため開発が活発に進んでいる。

oracleDB
→リレーショナルデータベース。言語はsql。oracle社が開発。

PostgreSQL
→リレーショナルデータベース。無料で利用でき、機能の改変や追加も自由。

SQLite
→同時接続を並列に処理する機構が比較的弱い。ローカルのテスト用としては十分に使える。

SQL Server
→企業で多く使われている。windowsで使われることが多くマウス操作ができ簡単。oracleよりコスパが良い

CRUD
→create(生成)read(読み取り)update(更新)delete(削除)のシステムに必要な4つの主要機能

トランザクション
→複数の処理を1つにまとめたものでそれらは分割できない。

コミット
→トランザクション処理で結果が確定すること。ファイルの変更履歴の保存やバージョン管理システムで結果を反映させること。

ロールバック
→トランザクション処理で処理した結果を取り消してやらなかったことにすること。

排他制御
→誰かが使ってる時に他の人が使えないようにする仕組み。

楽観ロック
→変更中に誰かに更新されていなければ変更し、更新されていた場合は変更を破棄する排他制御。

悲観ロック
→変更中はロックをかけ他の人に変更されないようにする排他制御。

実行計画
→いかに短い時間でsqlを実行できるか計算して導き出された方法。

チューニング
→システムやソフトウェアの設定を調整して使用状況に適した状態にすること。

パフォーマンス
→処理性能や実行速度、通信速度や容量のこと。

スロークエリ
→速度の遅いsql。具体的には2秒以上かかっている状態。

クエリログ
→クライアントからsqlへの接続情報、実行情報のログ。エラーが出た際に何が原因か把握するのに役立つ。

IDE
→テキストエディタ、コンパイラ、デバッガを1つにまとめたもの。統合開発環境ともいう。

Eclipse
→Javaのの開発環境が含まれているIDE。Javaの開発で使われることがおおい。

NetBeans
→Javaの開発で多く使われているIDE。Eclipseより初期設定が楽でGUIが見やすい。

SpringToolSuite
→EcripseをベースとしたSpring Frameworkを使うために開発されたツール。基本的な使い方はEclipseと同じ。

A5:SQL Mk-2
→データベース開発を支援するために開発されたフリーのSQL開発ツール。sqlの実行、テーブルの編集、ER図の作成などができる。

pgAdmin
→データベース開発支援のためのsql開発ツール。GUIで管理ができる。

DBeaver
→UIがEclipseと同じなので操作がしやすい。MysqlやPostgreSQLなど多くのDBに対応している。

MySQL Workbench
→MySQL公式から配布されている管理ツール。GUI操作のため使いやすい。

Sequel Pro
→Mac用のMySQLデータをGUI操作で管理できるツール。

TeamSQL
→多くのDBに対応している管理ツール。現在はサービス終了している。

PM（プロジェクトマネージャー）
→プロジェクトの運営、品質、納期等に責任を持つプロジェクト管理者。システム開発の一連の業務に精通していなければいけないためSE→PLとキャリアアップしていく。

PL（プロジェクトリーダー）
→プロジェクトにおけるサブチームのリーダーとして、メンバーの工数、進捗、品質などを管理する。

PG（プログラマー）
→SEが作成した設計書に沿ってプログラミングを行う仕事のことです。システム開発における製造工程を担当する。

SE（システムエンジニア）
→顧客の要求から仕様を決定し、大まかな設計をする。プログラマーのリーダー。

デザイナー
→クライアントや依頼者の求めに応じた顧客層と、機能性、コストなど、さまざまな条件を踏まえた上で最適なデザインを考案する

フロントエンド・エンジニア
→フロントエンド側を担当するエンジニア。ユーザーと直接データのやり取りを行う要素のことで、Web制作ではWebブラウザ側を指す。

サーバーサイド・エンジニア
→サーバーサイドを担当するエンジニア。Webサイトを操作して送られてくるデータを保管場所に保存するために処理したり呼び出したりする環境を作成し、メンテナンスを行ったりバージョンアップの際に対応したりする仕事。

バックエンド・エンジニア
→Webサーバー、データベースといったユーザーからは見えない裏側のシステム部分を構築する。サーバーサイドエンジニアと大きな違いはない。

UI
→ユーザーインターフェイス。見た目、使い勝手、画面のこと。

CUI
→キーボードでのみ操作する画面。

GUI
→マウスや指で操作できる画面。

ファイル
→コンピューター世界における書類のこと。読み込み、書き出し、編集などができる。

拡張子
→ファイル名のうち.の右側の部分。ファイルの種類を表す。

ディレクトリ
→コンピューター世界における箱のこと。ファイルを複数格納することができる。

ルート・ディレクトリ
→階層型ファイル構造における最上位（根、root）のディレクトリのこと。

カレント・ディレクトリ
→現在位置として指し示しているディレクトリのこと。

パス
→ファイルやフォルダの置いてある場所のこと。

絶対パス
→ルートディレクトリから目的地までの経路を表す。

相対パス
→ユーザが現在作業しているフォルダから目的のフォルダまでの経路を表す。

LAN
→コンピュータや通信機器、情報機器などをケーブルや無線電波などで接続し、相互にデータ通信できるようにしたネットワークのこと。

WAN
→広域のネットワークのことで、点在するLANとLANを結ぶ公衆網のこと。WANを世界規模で実現しているのがインターネットである。

クライアント
→サービスや機能を使う側のコンピュータや依頼主的なお客さまのこと。

サーバー
→サービスや機能を提供する側のコンピュータのこと。

HTTP
→WebサーバとWebクライアントの間でデータの送受信を行うために用いられる約束事。

FTP
→WebサーバとWebクライアントの間でデータの送受信を行うために用いられる約束事。ファイル転送のみを目的としている。

HTTPS
→Webのデータ転送に用いられるHTTPが暗号化されている状態を表したもの。盗聴や第三者によるなりすましを防止する。

SSL
→インターネット上でやり取りする情報を暗号化して送受信するための仕組み。

SSH
→ネットワークを介して別のコンピュータにログインして操作するためのソフトウェアの一つ。通信経路が暗号化されるため安全。

ポート、ポート番号
→ネットワークとパソコンの間にある識別用の番号のこと。データの種類ごとに番号が決まっている。

HTTP、FTP、SSHの各ポート番号
→HTTPは80。FTPは21。SSHは21。

ファイアウォール
→ネットワークとパソコンの境界に設置され、内外の通信を中継・監視し、外部の攻撃から内部を保護するためのソフトウェアや機器、システムなどのこと。

TCP
→安全性の高い通信方。相手が確実にデータを受け取ったかを確認したり、データの欠落や破損を検知して再送したりできる。

IPアドレス
→インターネットをするときにコンピュータに割り当てられる住所のこと。

グローバルIPアドレス
→インターネットに接続する機器が使用できるIPアドレス。

プライベートIPアドレス
→組織内のネットワーク（プライベートネットワーク）でのみ使用できるIPアドレス。

ドメイン
→インターネット上の住所のようなもので、ホームページアドレスやメールアドレスに含まれている。@のあとやwww.のあとの文字列を指す。

DNS
→インターネット上でドメイン名（ホスト名）とIPアドレスの対応関係を管理するシステム。ただの文字列から日常的で親しみやすい別名をつける。

VPN
→普通の電話線やインターネットの線を使いながらも専用線のような安全性の回線。

ブリッジ
→ネットワーク中のデータを見て、行き先を振り分けてくれる機器のこと。

プロキシサーバー
→企業の社内ネットワークなどからインターネットに接続する際に、出入り口で中継をしているサーバー。

SMTP
→WebサーバとWebクライアントの間でデータの送受信を行うために用いられる約束事。メール送信のみを目的とする。

POP
→WebサーバとWebクライアントの間でデータの送受信を行うために用いられる約束事。メール受信のみを目的とする。メールサーバからダウンロードする。

IMAP
→WebサーバとWebクライアントの間でデータの送受信を行うために用いられる約束事。メール受信のみを目的とする。メールサーバ上で管理、操作する。

セッション
→接続/ログインしてから、切断/ログオフするまでの、一連の操作や通信のこと。

クッキー
→Webサイトとユーザー間でやり取り・保存されるユーザーのアクセス情報のこと。ログインを省略できる。

キャッシュ
→インターネットブラウザで一度読み込んだ画像データなどをキャッシュファイルとして保存する情報のこと。ページの表示を早くする方法として利用されている。

インデックス
→索引とも呼ばれ、データベースへのアクセス効率を向上させるために、検索対象となる1つ以上の列(属性)に対して設定される仕組み。

フロー
→コンピュータプログラムを構成する命令や手続き、処理の流れなどのこと。

タスク
→ソフトウェアの実行単位のこと。

Git
→オープンソースの分散バージョン管理システムの一つ。複数の開発者が共同で一つのソフトウェアを開発する際などに、ソースコードやドキュメントなどの編集履歴を統一的に管理するのに用いられる。

SVN
→オープンソースの集中バージョン管理システム。サーバで管理しているソースリポジトリに対し、クライアントから作業コピーを受け取り編集して、またサーバに編集内容（差分）を送っている。

デグレ
→デグレードの略。新しいバージョンのソフトウェアの品質が、以前より悪くなること。また、以前修正した不具合やバグが再発・復活すること。

マージツール
→複数のファイルやデータ、プログラムなどを、決められたルールに従って一つに統合すること。
